### KL Divergence

衡量两个概率分布之间的相似性，两个概率分布越相近，KL散度越小。

![1616826197155](README.assets/1616826197155.png)

P 是真实分布，Q 是模拟分布，也就是说用 Q 去模拟 P。所以 D(P|Q) 和 D(Q|P) 含义是完全不同的。

假设某个班里男女比例分别为：[0.4, 0.6]

A 猜测比例为 [0.3, 0.7]

```
KL1 = (0.4*log0.4 - 0.4log0.3) + (0.6*log0.6 - 0.6log0.7)
	= 0.4*log1.3333 + 0.6log0.8571
	= 0.1151 - 0.0925
	= 0.0226
```

B 猜测比例为 [0.2, 0.8]

```
KL2 = (0.4*log0.4 - 0.4*log0.2) + (0.6*log0.6 - 0.6*log0.8)
	= 0.4*log2 + 0.6*log0.75
	= 0.2773 + -0.1726
	= 0.1047
```

KL1 < KL2，所以 A 的概率分布和真实概率分布更为接近，直觉上也是如此，A 猜的更准。

## 并查集

```cpp
class UF {
public:
    vector<int> fa;
    vector<int> sz;
    int n;
    int comp_cnt;
    
public:
    UF(int _n): n(_n), comp_cnt(_n), fa(_n), sz(_n, 1) {
        iota(fa.begin(), fa.end(), 0);
    }
    
    int findset(int x) {
        return fa[x] == x ? x : fa[x] = findset(fa[x]);
    }
    
    void unite(int x, int y) {
        if (sz[x] < sz[y]) {
            swap(x, y);
        }
        fa[y] = x;
        sz[x] += sz[y];
        --comp_cnt;
    }
    
    bool findAndUnite(int x, int y) {
        int x0 = findset(x);
        int y0 = findset(y);
        if (x0 != y0) {
            unite(x0, y0);
            return true;
        }
        return false;
    }
};

class Solution {
public:
    bool containsCycle(vector<vector<char>>& grid) {
        int m = grid.size();
        int n = grid[0].size();
        UF uf(m * n);
        for (int i = 0; i < m; ++i) {
            for (int j = 0; j < n; ++j) {
                if (i > 0 && grid[i][j] == grid[i - 1][j]) {
                    if (!uf.findAndUnite(i * n + j, (i - 1) * n + j)) {
                        return true;
                    }
                }
                if (j > 0 && grid[i][j] == grid[i][j - 1]) {
                    if (!uf.findAndUnite(i * n + j, i * n + j - 1)) {
                        return true;
                    }
                }
            }
        }
        return false;
    }
};
```

```cpp
class Solution {
public:
    bool containsCycle(vector<vector<char>>& grid) {
        int n = grid.size(), m = grid[0].size();
        vector<vector<int>> visited(n, vector<int>(m, -1));
        int dx[] = {1, -1, 0, 0};
        int dy[] = {0, 0, 1, -1};
        for (int i = 0; i < n; i++)
        {
            for (int j = 0; j < m; j++)
            {
                if (visited[i][j] == -1)
                {
                    queue<pair<int, int>> eq;
                    visited[i][j] = 0;
                    char ch = grid[i][j];
                    eq.push({i, j});
                    while (!eq.empty())
                    {
                        auto now = eq.front(); eq.pop();
                        for (int t = 0 ; t < 4; t++)
                        {
                            auto nextt = now;
                            nextt.first += dx[t]; nextt.second += dy[t];
                            if (nextt.first < 0 || nextt.first >= n || nextt.second < 0 || nextt.second >= m || grid[nextt.first][nextt.second] != ch) continue;
                            if (visited[nextt.first][nextt.second] == -1)
                            {
                                visited[nextt.first][nextt.second] = visited[now.first][now.second] + 1;
                                eq.push(nextt);
                            }
                            else
                                if ((visited[nextt.first][nextt.second]  - visited[now.first][now.second] ) != -1) return true;
                        }
                    }
                }
            }
        }
        return false;
    }
};
```

```cpp
class Solution {

    vector<vector<int>> vis1, vis2;

    bool dfs(int x, int y) {
        if (vis2[x][y]) {
            return true;
        }
        vis1[x][y] = vis2[x][y] = 1;
        if (y + 1 < n && grid[x][y] == grid[x][y + 1] && dfs(x, y + 1)) {
            return true;
        }
        if (x + 1 < m && grid[x][y] == grid[x + 1][y] && dfs(x + 1, y)) {
            return true;
        }
        return false;
```

1.**讲证据，多表格，可视化**

![img](README.assets/v2-98c318986c89aae63d04159c505dfc16_b.jpg)

讲证据，多表格，可视化是第一个比较重要的点。

**能用表格说明问题的，就不用文字，能用图说明问题的，就不用表格**。可视化呈现工作成果的重要性不用多说。

2.**深挖原因，对问题多重分析，给出建议**

第二个需要注意的点是深挖原因，对问题多重分析给出建议。

为什么这个很重要呢？

如果你是做新媒体运营的，你本周的文章打开率是5%，相比上周下降了2%（上周是7%）。

那么数据变差的原因是什么，是不是需要分析一下呢？

原因应该非常的多，但很多人可能就停留在摆出事实，没有去分析原因。

如果往下挖，可能会有很多种原因，文章标题、头图、主编请假、换了新主编等等，各种内部或外部的原因。

那么你可以把这些原因都一一列出来，这样你才能知道下一步该怎么做才能把文章的打开数据提升上去。

当你和领导都知道下一步该怎么做，就会产生很多**确定感**，团队也会有**安全感**，不会一无所知，不知道如何下手。

3.**写出来的东西要积极主动**

**只写“可以做，应该做”，不写“不可以做，做不到”**

第三个需要注意的点是在写作的过程中，你写出来的内容一定要**积极主动**，至少要显得你很积极主动。

那么给大家一个建议，报告内容只写什么可以做，什么应该做而不要去写不可以做，做不到。

因为如果你去写不可以做、做不到，会非常打击你自己和领导以及同事的信心，团队没有信心，就会觉得没有希望而消极怠工，这对整个团队的影响是非常大的。

所以领导们比较喜欢看到的是“可以做，应该做”，然后事情交给你，你可以做到。

**先“认同”，再“但是”**

即使你觉得事情有难度做不到，也不能直接说明，需要用委婉的方式表达出来——先认同再但是。

“先认同再但是”是一个我觉得非常好的表达格式。

比如最近有一个老师，在帮我做课，总是拖延，无法交付。每次跟他要PPT，都是说“很快”，但是等几天还是没有内容。

在报告中，我要怎样把这件事**委婉的说出来**呢，因为这个内容是我必须写的，但是又不能直接说这个老师不行，这个课难产或者已经坑了。

那么我会这么去写。

**首先，列出这个老师的所有优点**。

夸这个老师态度很好，积极主动，干货很多等等。

**然后，再写“但是”**。

这个老师工作非常忙，所以我们需要多花一些时间去磨合。

这样就很委婉的把这个老师拖延的问题表达出来了。

所以在工作中，我们需要多去想一想到底该怎么表达才能给别人的**观感**会更好。

**一、小白工作日志**
------------

> 例1：x月x日，x局在五楼会议室召开纪律学习教育月会议，各科室、直属机构全体人员参加。

**修改意见**：  
为深入开展纪律教育学习月活动**（事由）**，xx局于x月x日在局7楼会议室召开“以案为鉴”警示教育辅导会。会上，xx副局长**（人物）**结合重大工程建设领域的相关案例，就贪污受贿、失职渎职和廉洁从政等重要内容做了深入浅出的讲解**（措施）**，进一步增强了参会人员的遵纪守法意识和依法行政意识**（成效）**。xx局长**（人物）**结合辅导会内容，部署下一阶段党务廉政、队伍管理及执法管理等工作。

为什么一定要加上“事由和措施”呢?  
因为政府工作报告的基本框架就是如此：**事由+主体（措施）+结尾（成效）**。各级政府报告里肯定有一项内容是民生福祉工作成效。接下来就要简单罗列做的工作：一是扶贫工作的措施和成效。二是公共服务均等化工作的措施和成效。三是居民医疗事业的措施和成效……诸此种种。而且，如果没有事由，今后总结工作的时候就容易遗忘，到底是例行工作，还是上级直管部门安排工作，还是当地政府部署的工作任务？不容易迅速分辨。如果没有措施，等我们需要总结工作的时候，又需要花费额外的时间来整理思路，以便厘清我们到底通过什么措施来推进该项工作的。

> 例2：xx局纪律教育学习月活动工作总结的开篇为：按照省、市关于开展纪律教育学习月活动的要求，我局结合实际，从7月起，扎实开展“以案为鉴”警示教育活动，现将相关工作情况汇报如下……

**解析：分门别类，帮助记忆，方便查询和调取。**  
今天笔者分享的是多年工作实践中不断优化、改良的日志格式，但大家要明白——**日志格式没有最优版，只有最佳版**。适合自己特点、适合自身岗位的才是最好的格式。上面冗长的案例讲解只是想告诉大家，不同时期、不同工作、不同岗位的文秘其工作日志都需要适时调整，以适应不同工作和需求。在实际案例中，大部分领导在检查中必然会涉及到相当多的具体问题，要反馈下级的请示、汇报，接受管理对象和群众的意见建议等等。这些繁琐的事务其实都是今后工作材料中为各种观点提供支撑的宝贵素材。如果我们未雨绸缪，提前做好材料的分门别类和归纳整理工作，在撰写材料的时候就能像珠宝匠一样把这些零碎的珠宝串起来，让它们变废为宝。
